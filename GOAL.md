## 레퍼런스의 핵심 설계 언어 (관찰로 확정)

1. **무채색 무광 3D(흰/회색).** 오직 움직이는 성부 마커만 컬러. 발광·고채도 없음.
2. **음악의 "구조"를 3D 건축/조형으로 빚는다.** (디오라마 섬 / 거울 대칭 조형)
3. **성부 = 구조를 통과하는 작은 컬러 마커.** 캐논 = 같은 구조를 시차/거울로 통과.
4. **아이소메트릭 부감 + 부드럽고 명상적인 카메라.**

## 현재 구현 상태 (2026-06-21)

- 백엔드: music21 분석(성부/절대초/importance) + 캐논·화음·기타 분류.
- 프론트: Three.js **연속 스크롤 높이 지형 + 성부색/피치클래스색 + 블룸** + OSMD 악보 + Tone.js.
- → 미감·개념이 레퍼런스와 거의 정반대(채도·발광 과다, 연속 지형 vs 이산 건축).

## 비(非)캐논 입력 처리 — 구조적 대응 (요청 반영)

> 들어오는 곡이 캐논이 아닐 수 있다. 레퍼런스는 캐논 전용이지만, 본 앱은 **임의의 MIDI**를 받는다.
> 따라서 **분류 결과(캐논/화음/기타)에 따라 시각화 모드를 분기**하는 구조가 필요하다.

- **캐논**: 레퍼런스 그대로 — 같은 구조를 성부 마커가 시차로 통과(시차 추격) + 거울 대칭 옵션.
- **화음(다성, 비캐논)**: 시차 모방이 없으므로 "추격"은 부적절. 대신 **여러 성부가 한 구조를
  동시에 다른 경로로 통과**하거나, 화성 두께를 조형으로 표현(현 기둥 아이디어 재활용).
- **기타(단성)**: 성부 1개 — **단일 마커가 하나의 디오라마 길을 여행**. 가장 단순한 폴백.
- 공통: 미감·디오라마·마커 패러다임은 세 모드가 공유하고, **차이는 "성부 간 관계를 어떻게
  배치/강조하느냐"** 뿐이다. → 시각화 코어를 "구조 생성 + 마커 주행"으로 두고, 모드별 배치 전략만 교체.

## 확정 결정 (grill-with-docs, 2026-06-21)

> 스탠스(음높이=높이, 큐브가 음표 따라 주행)는 **유지**. 레퍼런스도 같은 패러다임이며, 차이는
> 미감 디테일 4가지뿐임을 프레임 분석으로 확인. 무채색은 캐논 부수효과라 색은 유지.

- **차이 4종을 각각 토글 옵션으로 채택** → 기존/변경 비교 가능 (ADR 0011):
  ① 이산 블록화(smooth|stepped) ② 발광 제거(glow|matte) ③ 디오라마 무대화(scroll|diorama)
  ④ 트랙 가독성(wide|narrow). 전부 기존값=현행, 전부 신규값=레퍼런스 느낌.
- **캐논/비캐논 동일 엔진** (ADR 0012): 분류로 엔진을 분기하지 않음. 캐논 강조(시차추격+거울)는
  캐논 분류 시에만 켜지는 가산 레이어. 비캐논은 강조만 꺼진 같은 엔진.
- **악보 패널**: OSMD(오선보) / 2D 피아노롤 선택형.
- 분류 오판(캐논곡→"화음") 보정은 ADR 0005 개정 과제로 연동.
