# Context

## Core Terms

- **성부**: 동시에 진행되는 독립적인 음악 줄기. 백엔드 분석 JSON의 `part`, 프런트엔드의 레인/성부와 대응한다.
- **캐논**: 한 성부의 주제가 다른 성부로 시간차를 두고 모방되는 구조.
- **화성**: 서로 다른 성부가 같은 순간에 함께 울리며 만드는 수직적 관계.
- **기타**: 단성에 가깝거나 위의 분류에 안정적으로 들어가지 않는 입력.
- **리듬 성부**: 음높이가 없는 타악기 성격의 성부.
- **피치 클래스**: 옥타브를 제거한 12음 체계의 클래스. 색상 매핑의 기준이다.
- **성부 중요도**: 렌더링과 상호작용 우선순위를 정하는 점수.
- **레인**: 3D 장면에서 성부 하나를 배치하는 z축 구획.
- **리본**: 성부의 음높이 진행을 따라가는 3D 지형 메쉬.
- **음의 길이**: 하나의 음이 지속되는 시간. 시간 축과 보이는 구간을 연결하는 기본 단위.
- **섬(Island)**: 음악 구조 단위. 선율 패턴이 동일하거나 유사한 구간들이 공유하는 3D 공간. 시간 구간(time window)이 아니라 음악적 내용(melodic content)이 경계를 결정한다. 같은 선율이 다른 시간에 재등장하면 같은 섬으로 귀속된다.
- **구조 단위(Structural Unit)**: 반복 감지의 기본 입력. 특정 음정열(interval sequence)이 일정 유사도 임계치 이상으로 반복될 때 하나의 구조 단위로 간주한다.
- **공유 지형(Shared Terrain)**: 하나의 섬 위에서 여러 성부가 함께 사용하는 단일 terrain mesh. 캐논에서는 선행 성부의 notes로 빌드되며, 후행 성부는 별도 terrain 없이 이 위를 시차(lagSec)로 달린다.
- **Z 유사도 배치**: 음악적 유사도가 높을수록 Z 거리가 가까워지는 레이아웃 원칙. 캐논(완전 동일) → Z 완전 공유. 화음(부분 유사) → 좁은 Z 클러스터. 무관 → 넓은 Z 분리.
- **순환 섬(Cycling Island)**: 큐브가 period마다 x=0으로 돌아오며 반복 순환하는 섬. 캐논에 적용. `cube.x = ((t - voiceStart) % period) * X_PER_SEC`.
- **주제 주기(Period)**: 순환 섬의 X축 길이를 결정하는 반복 단위 시간. 초기값 = `lagSec`, 자기 유사도 분석 후 구조 단위(structural unit) 감지 결과로 교체.
- **자기 유사도(Self-Similarity)**: 같은 성부 내 서로 다른 시간 구간의 음정열을 비교해 반복 패턴을 감지하는 분석. 섬 경계 및 주제 주기 결정의 근거.

## Source Of Truth

- 분석 결과는 백엔드가 생성한 JSON이다.
- 재생과 시각화는 공통 시간축을 공유한다.
- MusicXML은 악보 표시용 보조 표현이다.
