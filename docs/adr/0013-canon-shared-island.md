# ADR 0013 — 캐논 공유 섬 + 순환 횡단

## 상태
제안됨 (2026-06-23)

## 배경
grill-with-docs 세션(Q1~Q11)에서 아이디어 1·2·3을 검증하며 확정된 설계 결정들을 기록한다.

현재 엔진(`terrain.js`)은 각 성부가 독립된 laneZ와 terrain mesh를 가진다. C2(시차 추격)는
캐논 후행 성부의 큐브를 선행 성부의 laneZ로 이동시키지만, 후행 성부의 terrain mesh는 여전히
독립적으로 남아 있다. 이 결정은 그 불완전함을 해소하고 "캐논 = 같은 길, 시차"라는 시각 논리를
끝까지 밀어붙인다.

## 결정

### 1. 섬(Island) = 음악 구조 단위
섬의 경계는 시간 구간(10초 균등 분할, Phase B)이 아니라 선율 패턴의 반복으로 정의한다.
동일하거나 유사한 음정열(interval sequence)이 다른 시간에 재등장하면 같은 섬으로 귀속된다.
감지 방법: 자기 유사도(self-similarity) — 같은 성부 내 [t0, t1] vs [t2, t3] 음정열 비교.

### 2. 캐논 공유 지형 (Shared Terrain)
캐논 성부들은 하나의 terrain mesh를 공유한다.
- 지형은 선행 성부(leader)의 notes로 빌드된다.
- 후행 성부는 별도 terrain을 빌드하지 않는다.
- 모든 캐논 성부의 큐브는 같은 Z 위치(leader laneZ)에서 달린다.

### 3. 순환 횡단 (Circular Traversal)
캐논 큐브는 섬을 선형으로 통과하는 것이 아니라 period마다 순환한다.

```
cube.x_in_island = ((timeSec - voiceStartTime) % period) * X_PER_SEC
cube_world_pos = island_base_position + cube.x_in_island
```

period 결정 (단계적):
1. 초기: `lagSec` (백엔드 `canon.pairs[*].lagSec`)
2. H1(자기 유사도 분석) 완료 후: `structuralUnit.period`로 교체

### 4. Z 유사도 배치 (Z-Similarity Layout)
음악적 유사도가 높을수록 Z 거리가 가까워진다.

| 관계 | Z 배치 |
|---|---|
| 캐논 (완전 동일 선율) | Z 완전 공유 (laneZ = leader와 동일) |
| 화음·다중주 (부분 유사) | 좁은 Z 클러스터 (gap ≈ 1.5) |
| 무관한 성부 | 넓은 Z 분리 (기존 LANE_GAP 유지) |

### 5. 지형 고정, 큐브 높이 독립
지형(terrain)은 leader의 선율로 고정된다. 변주가 있는 경우 큐브의 y(음높이)는
실제 연주 음을 따르므로 지형에서 뜰 수 있다 — 이것이 변주를 시각적으로 표현하는 방법이다.
지형 모핑 애니메이션(업그레이드)은 별도 ADR로 분리 예정.

## 대안들

**선형 전개** — 큐브가 항상 앞으로만 진행(현재 모델). 구현 단순하지만 캐논의
"반복"이 공간에 드러나지 않는다.

**복사본 방식** — 반복 시마다 같은 모양의 새 섬 생성. 비캐논 형식 재방문(A-B-A,
두 구간이 시간적으로 겹치지 않음)에는 충분하지만, 캐논처럼 두 성부가 동시에
같은 섬 위에 있어야 하는 경우에는 부적합하다.
→ 비캐논 형식 재방문(H4)에는 복사본 방식을 사용한다.

## 결과
Phase H(H1~H5)로 구현 계획 수립. 자기 유사도 분석(H1)이 선행되어야 H2~H4가 가능하다.
C2(시차 추격)와 H2(순환 섬 렌더링)는 충돌 지점이 있으므로 H2 구현 시 C2 코드를 대체한다.
