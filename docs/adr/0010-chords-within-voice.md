# ADR 0010 — 성부 내 동시 화음 처리 (ADR 0003 결정 8 보강)

## 맥락
한 성부가 동시에 여러 음(화음)을 칠 때, 기존엔 최고음만 저장 → 소리·지형에서 안쪽 화음 음이
통째로 사라졌다(예: 피아노 왼손 3화음이 한 음처럼 들림/보임).

## 결정
- **데이터**: 음표마다 `chordMidis`(그 순간 동시 발음 전체, 정렬)를 보존. 단음은 `[midi]`.
- **소리**: 화음 전체를 재생(Tone.js PolySynth 에 주파수 배열 전달). 더 이상 최고음만 들리지 않음.
- **3D**: 큐브는 여전히 최고음 높이를 탄다. 추가로 그 구간에 **최저~최고음을 잇는 반투명 기둥**
  (InstancedMesh, 성부색 반투명)을 세워 화음 두께를 시각화.
- **악보**: OSMD/MusicXML 이 원래 화음을 그대로 표기(변경 없음).

## 근거
큐브 단선의 단순함은 유지하되, 소리·시각 모두에서 화성(harmony)을 복원. 기둥은 인스턴싱으로 성능 영향 최소.

## 대안
- 화음 음마다 작은 큐브 다수: 정밀하나 화면이 산만 → 기각.
- 최고음만 유지(현행): 화성 손실 → 기각.
