# ADR 0008 — 문서·테스트

## 맥락
결정이 많아 추적·변경이 필요. 비주얼 앱은 자동 검증이 까다로움.

## 결정
- **ADR 분리**: 구현 0단계에 결정들을 `docs/adr/*.md`(맥락/결정/근거/대안)로 분리 + `docs/glossary.md`.
- **테스트**:
  - 백엔드 단위테스트: canon/bwv66/melody 샘플 → 분류 라벨·성부 수·importance 검증.
  - E2E(Playwright): 파일 업로드 → 분류 라벨 표시 → 캔버스 렌더·재생 시작 자동 확인.

## 근거
ADR 분리로 의사결정 추적성 확보. 단위테스트로 분석 로직 회귀 방지, E2E로 핵심 흐름 보장.

## 대안
- 계획 파일 한 곳 유지: 커지면 찾기 힘듦 → 분리.
- 수동 확인만: 회귀 위험 → 단위 + E2E.
