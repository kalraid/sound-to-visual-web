# ADR 0002 — 타이밍 단일소스·동기화

## 맥락
악보 커서·지형·큐브·소리가 조금이라도 어긋나면 영상이 망가진다. 여러 파서/시계가 있으면 드리프트.

## 결정
- **단일소스**: 백엔드 분석 JSON(절대 `startSec`/`durSec` 포함)이 유일 진실. 프론트는 mid 재파싱 금지.
- **재생**: 그 음표 리스트를 Tone.js 신스로 스케줄 재생. `Tone.Transport.seconds`를 모든 렌더의 공통 시계로.
- **재생 제어**: 재생/일시정지/탐색(seek)/배속. seek 시 전체가 Transport 시계에 재동기.
- **변수 템포**: 백엔드가 템포맵으로 박→초를 사전계산(절대초). Tone Transport BPM 자동화 미사용.
- **레이턴시 보정**: Web Audio 출력 지연 보정용 고정 오프셋을 화면 시계에 적용(소리 기준 정렬).

## 근거
하나의 데이터·하나의 시계 = 어긋남 원천 제거. 절대초 사용으로 변수 템포도 단순 처리.

## 대안
- 프론트 @tonejs/midi 재파싱: 두 파서 드리프트 위험 → 기각.
- AudioContext.outputLatency 자동 보정: 브라우저 편차 → 고정 오프셋이 단순·충분.
