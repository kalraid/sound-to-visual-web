# ADR 0015 — 롤러코스터 모드: 공유 피치 나선 + 음 교차

## 상태
확정됨 (2026-06-24) — grill-with-docs 세션 결과.
구현 메모: 현재 `terrain.js`의 `pathMode="coaster"`(I1, 성부별 장식 나선)는 **v0 플레이스홀더**이며 본 ADR의 모델로 교체 대상.

## 배경
`GOAL.md`의 본래 목표는 NHK 캐논 영상(무채색 디오라마 건축) 재현이었다. 사용자가
"각 큐브가 출발지·이동지가 제각각이면서 뱅글뱅글 도는 롤러코스터" 방향을 새 선호로 제시했고,
이어 "트랙기반이라 한 건, 시작점이 서로 다르고 자기 트랙을 따라가되 **공통된 음 위치를 통해
서로 교차하는 구조**를 잡고 싶었다"고 정정했다. 즉 핵심은 *한 트랙 공유*가 아니라 **여러 트랙이
공통 음에서 만나는 교차(crossing)**다.

## 결정
롤러코스터 모드를 디오라마/레퍼런스 모드와 **동등한 두 1급 시각화 모드**로 둔다(같은 엔진, 토글).
기하 모델은 다음과 같다.

1. **공유 피치 나선(Pitch Helix)**: 모든 성부가 공유하는 단일 기하. 피치클래스 → 각도(12등분),
   옥타브 → 높이. 따라서 *같은 음 = 같은 공간 점*.
2. **시간 = 움직임의 매개변수(공간축 아님)**: 큐브 위치는 오직 현재 음이 결정. 시간은 음과 음
   사이를 나선 위에서 활공하게 하는 파라미터. → 두 성부가 같은 음을 가지면 **시각이 달라도** 같은
   점으로 수렴·교차한다(교차점 = 캡처하려는 구조).
3. **전역 회전 = lap**: 나선 전체가 천천히 회전하며 전진감을 만든다. 1회전 = 1 코드 진행 주기.
   회전 주기 소스 우선순위: ① 코드 진행 period(아래 option 2) → ② 마디/다운비트(F1 measures) →
   ③ 고정 기본값.
4. **교차 레이어 2종**: (기본) 공유 음-위치 교차 + (가산) 구조 비트(다운비트/카덴스) 수렴.
5. **캐논과의 연결**: 후행 성부의 음 경로 = 선행의 음 경로를 lagSec 지연한 것이므로, 같은 나선
   위에서 선행을 추격하며 반복적으로 교차한다 → 캐논의 본질이 교차로 드러난다.

## option 2 (화성 진행 분석)의 범위
원래 "파헬벨 캐논 코드 진행 매칭"으로 출발했으나, grilling에서 **템플릿 없는 반복 코드 주기
감지**로 확정. 임의의 반복 화성 순환과 그 period를 추출하는 것이 목표이며, 이 period가 롤러코스터
lap(전역 회전 주기)을 구동한다. 특정 파헬벨 템플릿 매칭은 채택하지 않음(브리틀·범용성 저하).

## 결과
- 시간을 공간축에서 뗀 결과, 기존 `_worldXZ`/리본/디오라마 좌표계와 별개의 좌표 변환이 필요.
  거울(C4)·코너반사(G2)·디오라마 무대는 이 모드에서 숨김.
- "lap" 개념(CONTEXT의 순환 섬)이 *축 방향 복귀*가 아니라 *전역 회전 1바퀴*로 재해석됨.
- option 2가 분류뿐 아니라 **period 추출**까지 책임지며, 그 출력이 시각(회전 속도)에 직결됨.
- 현재 I1 구현은 이 모델과 다르므로 교체 대상(TODO Phase I 참조).
