# ADR 0006 — 성부 선정·성능·캐싱·드럼

## 맥락
성부가 많은 곡(8성부+)은 화면·성능 부담. 긴 곡 재분석은 느림. 드럼은 음정이 없음.

## 결정
- **성부 수 상한**: 기본은 핵심 성부 N개만 레인/큐브, 나머지는 배경 지형으로 합침.
  사용자가 최대 표시 개수 슬라이더로 조절.
- **핵심 성부 선정(importance)**: 복합 점수 = (음 활동량/음표밀도) + (최상성·최하성 외성 가중)
  + (캐논이면 주제선율 보유). 높은 순 N개. `analyzer.py`가 계산해 JSON에 포함.
- **드럼/비음정 트랙**: 제외하지 않고 바닥 펄스 리듬 트랙으로(맨 아래 레인, 비트 마커 + 타격 펄스).
- **캐싱**: 업로드 파일 해시 기반 분석 JSON 캐시(`backend/cache/<hash>.json`).
- **성능 목표**: 일반 데스크탑 60FPS. 지형 청크 분할 + 인스턴싱 + 카메라 컬링.
- **곡 범위**: 긴 곡 + 템포 변화 지원.

## 근거
핵심 성부 위주로 화면을 단순화하되 사용자 조절 여지. importance로 "무엇이 핵심인가"를 정량화.

## 대안
- 전부 표시: 많으면 FPS 저하 → 상한 + 슬라이더.
- 드럼 제외: 리듬감 손실 → 바닥 리듬 트랙으로 표현.
- 캐시 없음: 긴 곡 매번 대기 → 해시 캐시.
